微服务

2019-11-18 / 1 min read

微服务的定义

  • 每个微服务可以独立运行在自己的进程里
  • 独立运行的微服务共同构建整个系统
  • 每个微服务为独立的业务开发，一个微服务只关注某个特定的功能

微服务的特性

  • 可以使用不同的语言和数据存储技术
  • 微服务之间通过轻量的通信机制进行通信，例如REST API 进行调用
  • 全自动的部署机制（基石）

微服务全景架构图

微服务的优点

  • 单个服务更加容易开发、维护
  • 单个微服务启动快
  • 局部修改容易部署
  • 技术栈不受限（架构升级替换成本小）
  • 按需伸缩（根据微服务升级所需配置）

微服务的缺点

  • 运维要求高（需要全自动部署）
  • 分布式固有的复杂性
  • 重复劳动（可使用maven在java下共用工具类）

微服务的适用场景

  • 大型、复杂的项目
  • 有快速迭代的需求
  • 访问压力大

不适用微服务的场景

  • 业务稳定
  • 迭代周期长
